# Network Security Fundamentals
Network security protects data as it moves across an organization's infrastructure. Core controls include firewalls that filter traffic according to rules, network segmentation that isolates sensitive systems into separate zones, and virtual private networks (VPNs) that encrypt remote connections over untrusted networks. The defense-in-depth principle layers multiple independent controls so that the failure of any single mechanism does not compromise the entire network. Network access control (NAC) further restricts which devices may join the network based on their identity and security posture.

# Firewalls and Next-Generation Firewalls
A firewall enforces a security policy by allowing or denying traffic between network zones. Traditional packet-filter firewalls inspect source and destination addresses and ports, while stateful firewalls also track the state of active connections. Next-generation firewalls (NGFW) add deep packet inspection, application awareness, and integrated intrusion prevention, so they can block traffic based on the application or user rather than just the port. A web application firewall (WAF) is a specialized control that filters HTTP traffic to protect web applications from attacks such as SQL injection and cross-site scripting.

# Cryptography and Encryption
Cryptography secures information by transforming readable plaintext into unreadable ciphertext. Symmetric encryption such as AES uses one shared secret key for both encryption and decryption and is fast enough for bulk data. Asymmetric encryption such as RSA and elliptic-curve cryptography uses a public/private key pair, which enables secure key exchange and digital signatures. Hash functions like SHA-256 produce fixed-length fingerprints used to verify integrity; unlike encryption, hashing is one-way and cannot be reversed. Strong cryptography depends on protecting the keys, not the secrecy of the algorithm.

# Public Key Infrastructure and TLS
Public key infrastructure (PKI) is the system of certificate authorities, digital certificates, and policies that binds public keys to verified identities. A certificate authority (CA) signs certificates that clients trust, enabling secure identification of servers and users. Transport Layer Security (TLS) uses these certificates to establish encrypted, authenticated channels for web, email, and API traffic; it replaced the older, insecure SSL protocol. The TLS handshake negotiates cipher suites, verifies the server certificate, and derives session keys before any application data is exchanged.

# Authentication and Multi-Factor Authentication
Authentication verifies who a user is. Multi-factor authentication (MFA) strengthens login by combining something you know (a password), something you have (a hardware token or phone), and something you are (a biometric). MFA dramatically reduces account takeover because a stolen password alone is no longer sufficient. Phishing-resistant methods such as FIDO2 and hardware security keys defeat credential-replay attacks that can bypass one-time codes. Passwordless authentication uses cryptographic keys stored on a device instead of a shared secret.

# Authorization and Access Control
Authorization determines what an authenticated user is allowed to do. The principle of least privilege grants each user only the access strictly needed for their job, limiting the damage of a compromised account. Role-based access control (RBAC) assigns permissions to roles rather than individuals, which simplifies administration at scale, while attribute-based access control (ABAC) makes decisions from attributes such as department, location, and device posture. Privileged access management (PAM) adds extra protection, monitoring, and just-in-time elevation for high-risk administrative accounts.

# Identity and Access Management
Identity and access management (IAM) is the discipline of managing digital identities and their entitlements across an organization's systems. Single sign-on (SSO) lets users authenticate once and access many applications, reducing password fatigue and centralizing control. Federated identity standards such as SAML and OpenID Connect allow trust to be extended between organizations and cloud providers. Lifecycle processes for joiners, movers, and leavers ensure that access is granted, adjusted, and revoked promptly, preventing the accumulation of orphaned accounts.

# Malware and Threats
Malware is malicious software designed to damage systems or gain unauthorized access. Viruses attach themselves to legitimate files, worms self-propagate across networks without user action, and trojans disguise themselves as benign programs. Rootkits hide deep in the operating system to evade detection, and fileless malware runs entirely in memory using legitimate system tools. Modern endpoint detection and response (EDR) tools use behavioral analysis to catch novel malware that traditional signature-based antivirus would miss.

# Ransomware
Ransomware encrypts a victim's files and demands payment, usually in cryptocurrency, for the decryption key. Modern campaigns practice double extortion: attackers exfiltrate sensitive data before encrypting it and threaten to publish it if the ransom is not paid. Ransomware often enters through phishing, exposed remote desktop services, or unpatched vulnerabilities, then spreads laterally before triggering encryption. The most effective defenses are tested offline backups, rapid patching, network segmentation, and least-privilege access that limits how far an intrusion can spread.

# Social Engineering and Phishing
Social engineering attacks manipulate people rather than exploiting technical flaws. Phishing emails impersonate trusted senders to trick victims into revealing credentials, spear phishing targets specific individuals with personalized lures, and business email compromise (BEC) impersonates executives or vendors to authorize fraudulent payments. Because these attacks exploit human psychology rather than software bugs, technical controls alone are insufficient, and regular security-awareness training combined with reporting tools becomes a critical line of defense.

# Vulnerability Management
Vulnerability management is the continuous process of identifying, prioritizing, and remediating weaknesses in systems and software. Scanners enumerate known flaws, which are cataloged as Common Vulnerabilities and Exposures (CVE) entries and scored by severity using the Common Vulnerability Scoring System (CVSS). Because organizations cannot patch everything at once, teams prioritize using exploitability and business context, increasingly guided by whether a vulnerability appears in the CISA Known Exploited Vulnerabilities catalog. Patch management then deploys fixes in a controlled, tested manner.

# The OWASP Top 10 and Web Application Security
The OWASP Top 10 is a widely used awareness document listing the most critical web application security risks. It includes injection flaws such as SQL injection, broken access control, cryptographic failures, and security misconfiguration. SQL injection occurs when untrusted input is concatenated into a database query, allowing attackers to read or modify data; the defense is parameterized queries and input validation. Cross-site scripting (XSS) injects malicious scripts into pages viewed by other users and is mitigated by output encoding and a strict content security policy.

# Zero Trust Architecture
Zero trust abandons the old assumption that everything inside the network perimeter can be trusted. Its guiding motto is "never trust, always verify." Every request is authenticated, authorized, and encrypted regardless of whether it originates inside or outside the network. Micro-segmentation limits an attacker's lateral movement, and continuous verification repeatedly re-evaluates trust based on device posture, user behavior, and context throughout a session rather than only at login. Identity becomes the new perimeter in a zero trust model.

# Intrusion Detection and Prevention
An intrusion detection system (IDS) monitors network or host activity and raises alerts on suspicious behavior, while an intrusion prevention system (IPS) goes further and actively blocks malicious traffic. Signature-based detection matches activity against a database of known attack patterns, whereas anomaly-based detection flags deviations from a learned baseline of normal behavior. Anomaly detection can catch previously unseen attacks but tends to generate more false positives that analysts must triage. Network detection and response (NDR) extends these ideas with analytics across east-west traffic.

# Security Operations Center
A Security Operations Center (SOC) is the team and facility responsible for continuous monitoring and defense of an organization. Analysts rely on a Security Information and Event Management (SIEM) platform to aggregate logs from across the enterprise, correlate related events, and surface prioritized alerts. Tier 1 analysts triage alerts, Tier 2 analysts investigate deeper, and Tier 3 analysts handle complex incidents and threat hunting. SOC effectiveness is commonly measured by the mean time to detect (MTTD) and the mean time to respond (MTTR) to security incidents.

# SIEM and SOAR
A Security Information and Event Management (SIEM) platform collects, normalizes, and correlates log data from across an enterprise to detect threats and support investigations. Security Orchestration, Automation, and Response (SOAR) builds on the SIEM by automating repetitive response tasks through playbooks, such as enriching an alert with threat intelligence, isolating a host, or disabling an account. Together they reduce analyst fatigue and shorten response times, but they require careful tuning to control false positives and the cost of ingesting high-volume log sources.

# Incident Response
Incident response is the structured process organizations follow when a security breach occurs. The widely adopted NIST framework defines four phases: preparation; detection and analysis; containment, eradication, and recovery; and post-incident activity. Containment isolates affected systems to stop the threat from spreading, eradication removes the malicious presence, and recovery restores normal operations. A post-incident review then captures lessons learned so future responses improve. A well-rehearsed incident response plan and playbooks are essential before a crisis occurs.

# Digital Forensics
Digital forensics is the practice of collecting, preserving, and analyzing electronic evidence in a way that maintains its integrity for investigations or legal proceedings. Investigators capture forensic images of disks and memory, document a strict chain of custody, and work from copies to avoid altering the original evidence. Analysis reconstructs the timeline of an attack, identifies what data was accessed, and supports attribution. Volatile data such as running processes and network connections must be captured first because it disappears when a system is powered off.

# Threat Intelligence
Threat intelligence is evidence-based knowledge about adversaries, their motivations, and their methods that helps defenders make better decisions. Strategic intelligence informs leadership about risk trends, while tactical and operational intelligence provides indicators of compromise (IOCs) such as malicious IP addresses, domains, and file hashes that feed detection tools. Intelligence is most valuable when it is timely, relevant, and actionable, and when it is shared through communities and standards such as STIX and TAXII that allow organizations to exchange data efficiently.

# The MITRE ATT&CK Framework
MITRE ATT&CK is a globally accessible knowledge base of adversary tactics and techniques based on real-world observations. Tactics represent the attacker's goals, such as initial access, persistence, privilege escalation, lateral movement, and exfiltration, while techniques describe the specific methods used to achieve each goal. Defenders use ATT&CK to map their detection coverage, identify gaps, and communicate about threats with a shared vocabulary. It complements the Cyber Kill Chain by providing a far more granular and continuously updated catalog of behaviors.

# Threat Hunting
Threat hunting is the proactive search for adversaries who have evaded existing security controls, rather than waiting for an alert to fire. Hunters form hypotheses, often informed by threat intelligence or the MITRE ATT&CK framework, and then query logs, endpoint data, and network telemetry to confirm or refute them. Effective hunting assumes a breach has already occurred and looks for subtle signs of malicious behavior, such as unusual parent-child process relationships or anomalous authentication patterns. Successful hunts are converted into automated detections.

# Penetration Testing and Red Teaming
Penetration testing is an authorized, simulated attack that evaluates the security of systems by attempting to exploit vulnerabilities the way a real adversary would. A penetration test typically has a defined scope and reports exploitable findings with remediation advice. Red teaming is broader and more adversarial: a red team emulates a specific threat actor's tactics to test an organization's detection and response over an extended engagement. The blue team defends, and a purple team exercise has both sides collaborate to improve controls in real time.

# Endpoint Security and EDR
Endpoint security protects laptops, servers, and mobile devices, which are frequent entry points for attackers. Traditional antivirus relies on signatures of known malware, while endpoint detection and response (EDR) continuously records endpoint activity and uses behavioral analytics to detect and investigate suspicious actions. Extended detection and response (XDR) correlates this endpoint telemetry with network, email, and cloud signals for a unified view. Application allowlisting and host-based firewalls further reduce the attack surface on each device.

# Cloud Security
Cloud security operates under a shared responsibility model: the cloud provider secures the underlying infrastructure, while the customer remains responsible for securing their own data, identities, and configurations. Misconfigured storage buckets and overly permissive identity policies are among the leading causes of cloud data breaches. Cloud security posture management (CSPM) tools continuously audit configurations against best practices, and cloud workload protection platforms secure the running compute. Identity, not the network perimeter, becomes the primary control plane in the cloud.

# Container and Kubernetes Security
Containers package applications with their dependencies, and Kubernetes orchestrates them at scale, but both introduce new security concerns. Container images should be built from minimal, trusted base images and scanned for vulnerabilities before deployment. At runtime, the principle of least privilege applies: containers should not run as root, and Kubernetes role-based access control plus network policies should restrict what each workload can do and reach. Securing the software supply chain, including image registries and the CI/CD pipeline, is essential because a single poisoned image can compromise many clusters.

# Data Protection and DLP
Data protection ensures the confidentiality, integrity, and availability of information throughout its lifecycle. Data classification labels information by sensitivity so that the right controls apply, and encryption protects data both at rest and in transit. Data loss prevention (DLP) tools monitor and block the unauthorized movement of sensitive data, such as an employee emailing a customer database to a personal account. Strong key management, access controls, and retention policies round out a defense that limits both accidental leaks and deliberate exfiltration.

# Backup and Disaster Recovery
Reliable backups are the last line of defense against ransomware, hardware failure, and human error. The widely cited 3-2-1 rule recommends keeping three copies of data on two different media with one copy stored offsite or offline. Backups must be tested by performing actual restores, because an untested backup may silently fail when it is needed most. Disaster recovery planning defines the recovery time objective (RTO) and recovery point objective (RPO) that determine how quickly and to what point in time systems must be restored after a major incident.

# Security Governance and Compliance
Security governance aligns an organization's security program with its business objectives and risk appetite through policies, standards, and oversight. Risk management identifies, assesses, and treats risks by avoiding, mitigating, transferring, or accepting them. Compliance frameworks and regulations shape requirements: the NIST Cybersecurity Framework organizes activities into govern, identify, protect, detect, respond, and recover; ISO 27001 certifies an information security management system; and regulations such as GDPR, HIPAA, and PCI DSS impose specific obligations on personal, health, and payment data respectively.

# Distributed Denial-of-Service Attacks
A denial-of-service attack aims to make a system or service unavailable to legitimate users. A distributed denial-of-service (DDoS) attack amplifies this by using many compromised machines, often a botnet, to flood the target with traffic. Volumetric attacks saturate bandwidth, protocol attacks exhaust connection state, and application-layer attacks overwhelm specific services with seemingly legitimate requests. Defenses include upstream traffic scrubbing, rate limiting, content delivery networks that absorb load, and cloud-based DDoS mitigation services that filter malicious traffic before it reaches the origin.
